\chapter{Experiments}\label{chap:experiments}

This chapter provides empirical validation of the theoretical claims developed in Chapters~\ref{chap:centralized}--\ref{chap:marl}. We first describe some core implementation details that simplify our implementation of the MMATPG as a finite, formula-indexed simulation environment with a GPT-based MARL training pipeline. We then present five experiments, each validating a core theoretical result:

\begin{enumerate}
    \item \textbf{Centralized scheduling.} RLS achieves the predicted $(2-1/m)$ approximation ratio. (\S\ref{sec:exp-centralized})
    \item \textbf{Publication externality and market internalization.} Credit-based rewards produce unbounded PoA; the contract market eliminates the externality. (\S\ref{sec:exp-externality-market})
    \item \textbf{MARL convergence and equilibrium quality.} PSRO+MAPPO converges to approximate CCE with bounded PoA. (\S\ref{sec:exp-marl})
    \item \textbf{Ablation.} Each framework component contributes measurably. (\S\ref{sec:exp-ablation})
    \item \textbf{Scaling.} Theoretical bounds hold across instance sizes. (\S\ref{sec:exp-scale})
\end{enumerate}


\subsection{Implementation Details}\label{sec:implementation}

We implement a finite specialization of the MMATPG that preserves the core dynamics while making simulation and learning tractable. We describe the key design choices here; full implementation details (hyperparameters, training schedules, observation encoding) are deferred to Appendix~\ref{app:implementation}.


\subsubsection{Formula Universe and Latent Graph}

The formula space $\mathcal{F}$ is a finite set of $2n$ integer-indexed formulas, organized as $n$ theorem pairs. For each theorem $k\in\{0,\ldots,n{-}1\}$, formula $k$ and formula $k+n$ are negation partners: $\neg(k) = k+n$ and $\neg(k+n) = k$. A latent truth assignment $T:\{0,\ldots,n{-}1\}\to\{0,1\}$ selects one formula per pair as true; false formulas can be attempted but the proof kernel assigns them zero success probability.

Each instance is equipped with:
\begin{itemize}
    \item A difficulty map $d:\mathcal{F}_\top\to(0,1)$, governing base proof rates.
    \item A dependency DAG over true formulas, giving the unique latent $\delta^*$.
    \item A utility graph $G_U = (\mathcal{F}_\top, E_U, w)$ with edge weights $w(\psi,\phi)\in(0,1]$, where $w(\psi,\phi)=1$ for hard dependencies ($\psi\in\delta^*(\phi)$) and $w(\psi,\phi)<1$ for soft support (lemmas that help but are not strictly required).
\end{itemize}


\subsubsection{Sequent Compression}

To prevent exponential blowup in the action space, we collapse sequents to formula targets under a canonical local context. When agent $\alpha$ targets formula $\phi$, the sequent is automatically constructed as $[\mathcal{R}^\alpha_t \proves \phi]$, where $\mathcal{R}^\alpha_t$ is the set of formulas $\alpha$ has currently resolved. This induces a bijection between concrete formulas and concrete sequents, and ensures that all resolved sequents are automatically fully resolved (since antecedents contain only already-resolved formulas).


\subsubsection{Proof Kernel}

Proof outcomes are Bernoulli with deterministic dependencies. For agent $\alpha$ attempting $\phi$ with effective cumulative budget $\tau_{\mathrm{eff}}$ (including prior accumulated effort $\tau_{\mathrm{prev}}$), the success probability is:
\[
p^\alpha_{\mathrm{proof}}(\phi, \tau_{\mathrm{eff}}) = g(\phi)\cdot\Big(1 - \exp\!\big(-\rho^\alpha(\phi)\cdot(\tau_{\mathrm{eff}}^\kappa - \tau_{\mathrm{prev}}^\kappa)\big)\Big),
\]
where $g(\phi) = \mathbf{1}[T(\phi)=1]\cdot\mathbf{1}[\delta^*(\phi)\subseteq\mathcal{R}^\alpha_t]$ is a feasibility gate (false or dependency-incomplete formulas have zero probability), $\kappa\in(0,1]$ governs diminishing returns, and the effective rate is:
\[
\rho^\alpha(\phi) = e^{-\lambda_{\mathrm{diff}}\cdot d(\phi)}\cdot\Big(\rho_0^\alpha + \rho_1^\alpha\cdot\log\!\big(1 + \textstyle\sum_{\psi\in\mathcal{R}^\alpha_t} w(\psi,\phi)^\alpha\big)\Big),
\]
combining difficulty modulation with utility-weighted support from the agent's resolved library. On success, the dependency set is deterministically $\delta^*(\phi)$.


\subsubsection{Conjecture Kernel}

Conjecture success probability has the same conditional Weibull shape as the proof kernel, with a rate driven by the local ``opportunity mass'' over ghost formulas. On success, a formula is sampled from a softmax proposal distribution over the current ghost set, with selectivity increasing in budget (larger budgets yield higher-quality conjectures). The proposed formula and its negation become concrete in the conjecturing agent's private library.


\subsubsection{Query Model}

We use an oracular query model that has access to the true graph structure, truth assignment, and difficulty map, but adds persistent per-formula noise (logit noise on probability estimates, multiplicative noise on time estimates) to simulate imperfect self-assessment. The oracle returns $(\hat p, \hat\tau)$ snapped to the discrete budget grid. This is a modeling convenience; future work may study the effects of different query models (e.g., learnable predictors, noisy introspection) on learning dynamics and equilibrium quality.


\subsubsection{Market}

The market is the collateralized bilateral contract market defined in Chapter~\ref{sec:decentralized}. One implementation detail: accepting an offer transfers the position but does \emph{not} transfer cash at accept time. Cash flows occur only at settlement. This differs from the theoretical formulation (which specifies accept-time cash transfer) but is economically equivalent under full collateralization, since the collateral constraint ensures solvency at settlement regardless.

An exogenous bounty agent (not controlled by any learned policy) seeds the market at initialization by posting long offers on all initially concrete formulas at a fixed price, funded by a dedicated cash allocation. This provides baseline liquidity and ensures that learning agents face nontrivial market opportunities from the first episode.

Settlement triggers when: (i) the target formula is publicly resolved, (ii) the target's negation is publicly resolved (making the target impossible), or (iii) the deadline is reached. Long positions profit in case (i); short positions profit in cases (ii) and (iii).


\subsubsection{Instance Generation}

We use a synthetic DAG generator that produces layered, clustered dependency graphs with tunable depth $d$, branching factor $b$, and number of theorems $n$. Foundation nodes (shared prerequisites) are included to create realistic cross-cluster dependencies. Difficulty is derived from rarity scores (nodes with fewer dependents are harder), and utility weights are computed from two-hop dependency overlap.


\subsubsection{Policy Architecture}

Each agent is parameterized by a GPT-style causal transformer that processes a variable-length token sequence comprising:
\begin{itemize}
    \item \textbf{Formula tokens}: one per concrete formula, with features encoding resolved status, difficulty, dependency count, active job indicators, cumulative effort, and query estimates.
    \item \textbf{Market tokens}: one per outstanding offer and held position, with contract parameters.
    \item \textbf{Scalar token}: cash, worst-case balance, job status, timestep.
\end{itemize}

A set-attention encoder produces per-formula and summary embeddings, which are fed into a causal temporal transformer over recent history (context length $8$). The policy output is factored into: action-type head (categorical over $\{\mathrm{prove}, \mathrm{conj}, \mathrm{pub}, \mathrm{qry}, \mathrm{create\_post}, \mathrm{accept}, \mathrm{cancel}, \bot\}$), formula-pointer head (attention over formula tokens), and parameter heads (budget, deadline, price, side, quantity). Inadmissible actions are masked to zero probability.

The critic shares the encoder architecture but outputs a scalar value estimate. Under CTDE, the critic receives the same local observation as the actor (we do not use a privileged global-state critic in the current implementation).


\subsubsection{Reward}

The training reward combines realized one-step cash delta with optional shaping:
\[
r^\alpha_t = \underbrace{(c^\alpha_{t+1} - c^\alpha_t)}_{\text{economic}} + \underbrace{\sum_k \lambda_k\cdot\mathrm{shape}_k(s_t, a^\alpha_t)}_{\text{shaping}},
\]
where shaping terms include bonuses for successful proofs, publications that trigger settlement, and penalties for idling. Unsettled positions are \emph{not} marked to market in the reward; their value is realized only upon settlement. Rewards are normalized by initial bounty notional and number of theorems.


\subsubsection{Training}

Training uses PPO with GAE advantages, clipped surrogate objective, entropy regularization (linearly annealed), and AdamW optimization. The curriculum proceeds through five difficulty levels (4, 8, 16, 32, 64 theorem pairs with fixed number of agents), progressively shifting probability mass toward harder instances while retaining easier examples for stability. Fresh graph instances are sampled before each rollout.


\subsection{Experimental Setup}\label{sec:exp-setup}

\paragraph{Instance parameters.}
Unless otherwise noted, experiments use $m=2$ agents, synthetic DAG instances with depth $d=3$, branching factor $b=2$, geometric processing times ($\kappa=0.9$), oracle query model, loss parameter $\ell = 0.1$, and horizon $H=50$.

\paragraph{Baselines.}
\begin{itemize}
    \item \textbf{OPT}: Exact centralized optimum via dynamic programming (feasible for $n\le 16$).
    \item \textbf{RLS}: Reactive List Scheduling with shortest-expected-processing-time (SEPT) priority.
    \item \textbf{Greedy credit}: Each agent greedily maximizes credit-based reward; no market, no publication of byproducts.
    \item \textbf{Scripted market}: Hand-designed strategy: accept all available long bounties, prove highest-value available target, publish immediately upon resolution.
    \item \textbf{MARL}: Full framework (MAPPO + population-based training + curriculum).
\end{itemize}

\paragraph{Metrics.}
All metrics averaged over $N_{\mathrm{eval}} = 50$ rollouts with $95\%$ confidence intervals.
\begin{itemize}
    \item $\mathrm{SC}$: Expected makespan. \quad $\mathrm{PoA} := \mathrm{SC}/\mathrm{OPT}$.
    \item $\varepsilon\text{-exploit}$: Maximum unilateral improvement (via MAPPO fine-tuning of a deviator against the fixed joint policy). Measures distance to CCE.
    \item $\mathrm{Pub\%}$: Fraction of resolved formulas that are published.
    \item $\mathrm{Resolve\%}$: Fraction of target formulas that are fully resolved by horizon.
\end{itemize}

\subsection{Externality and Market Internalization}\label{sec:exp-externality-market}

\textbf{Claims:} (i) Credit-based rewards produce publication externalities with PoA growing in instance size (Theorems~\ref{thm:externality},~\ref{thm:unbounded-poa}). (ii) The contract market eliminates the externality and bounds PoA (Theorems~\ref{thm:internalization},~\ref{thm:bounded-poa}).

This experiment tests both claims simultaneously by comparing the MATPG (no market) and MMATPG (with market) on the same instances.

\textbf{Setup.} Two sub-experiments on the same instance family ($m=2$, $n=8$, $d=3$):
\begin{itemize}
    \item Externality severity: Vary cross-agent dependency density $\rho\in\{0.0, 0.25, 0.5, 0.75, 1.0\}$ (fraction of formulas whose full resolution requires a formula privately resolved by another agent). Run greedy-credit agents (MATPG) and scripted-market agents (MMATPG). Measure $\mathrm{Pub\%}$, $\mathrm{Resolve\%}$, and $\mathrm{PoA}$.
    \item PoA vs.\ loss parameter $\ell$. Fix $\rho=0.5$. Vary $\ell\in\{0.0, 0.05, 0.1, 0.2, 0.5\}$. Run scripted-market agents. Measure $\mathrm{PoA}$.
\end{itemize}
